Timing Analysis

  • The running time of the algorithm is O(E * f), where E is the number of Edges, and f is the value of the maximum flow. The FordFulkerson algorithm uses a BFS to find an augmenting path. Then the bottleneck residual capacity is calculated along the path, and the flow on each of the edges is increased/decreased by that amount.

  • In order to use the MaxFlow algorithm to solve the Bipartite matching, we add an artificial source and sink. We add an edge from source to all nodes in the first bipartite set, and an edge from all the nodes in the second bipartite set to the sink. We retain the normal edges in the original graph. And all the edges in this flow network are made to have capacity 1. In the Bipartite matching problem, the flow is bounded by V / 2, and hence the complexity can also be written as O(E * V).

Assumptions

  • The program expects certain command-line arguments to be provided for it to run correctly.

    Program Usage -
        ./ff problem graphFile [resultsFile]

        problem - max_flow or bipartite_matching
        graphFile - file containing information about graph
        resultsFile - optional file for writing back results

  • The format of the graph file for Task 1 is as follows -

    The first line contains two space-seperated integers V, E where V represents the number of vertices in the flow network, and E represents the number of edges.

    All the following lines have three space-seperated integers x, y, cap representing an edge from the vertex x to vertex y with capacity cap.

    It also assumes that there is exactly one node with indegree 0 (source), and exactly one node with outdegree 0 (sink).

  • The format of the graph file for Task 3 is as follows -

    The first line contains three space-seperated integers n1, n2, E where n1 represents the number of vertices in the first bipartite set, n2 represents number of vertices in the second bipartite set and E represents the number of edges.

    Each following line contains two space-seprate integers x, y representing an edge from vertex x to vertex y.

    The vertex numbers in each column are independend. Number 1 in first column and number 2 in second column represent two different vertices.

Coding Issues

  • The residual graph of a flow network is undirected. Usually an undirected edge x - y, is implemented as two directed edges x -> y and y -> x in adjacency list representation. But, here, the edges needed to be dependent on each other. Since a change in flow of one, must be reflected on the other. This meant instead of storing edges indepdently, we store a common pointer to them in both adjaceny lists.
